Le variabili

Servono per memorizzare dei valori

Ogni variabile sarà identificata da un nome che segue alcune regole:

  • niente spazi e niente trattini -, niente caratteri speciali (? . ecc)
  • Deve iniziare con una lettera, sono ammessi anche underscore e dollaro
  • Sono case sensitive
  • Non può iniziare con un numero
  • Il nome della mia variabile non deve mai coincidere con le keyword del linguaggio (if, else, switch, return. true, false)

Dichiarazione implicita delle variabili

Non è obbligatorio dichiarare le variabili, cioè fare un'operazione esplicita di creazione
quando c'è l'identificatore = l'engine crea implicitamente la variabile se non esiste già

Strict mode

e' un'opzione di autocontrollo del codice, ecmaScript 5 -> se viene dichiarat lo script mode js viene eseguito osservando le regole di scrittura più strette -> es. ci arriva segnalazione di errore se non dichiariamo le variabili mell'esecuzione normale js ci dà delle regole più permissive che se dichiariamo lo strict mode ci obbligano a uno stile di programmazione più strutturato

Var, let, const

var è la prima keyword usata da js per definire una variabile, var è accessibile in qualunque parte del mio script (globale)

let introdotta da EcmaScript è accessibile solo in un determinato ambito (scope) che è delimitato da un blocco di codice

un blocco di codice js è una serie di istruzioni delimitate da { .. blocco di codice .. }

const

Introdotte da EcmaScript6 -> servono a instruire l'engine che il valore assegnato con const non è modificabile
